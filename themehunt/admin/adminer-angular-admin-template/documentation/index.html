How to Run It?

You'll need to put the dist Folder on a server to run it. So you can upload the "dist" folder to your server to run it. Or you can build a server locally.

You can build a locally server using WAMP or MAMP (if you uses PHP). Also it's pretty easy to build one with Python or Ruby (if you prefer Python or Ruby). And here we'll build a local server with NodeJS

Preparation: You'll need to install Node.js (NPM comes along with it) and node-static (an NPM package)

Steps:

It's easy if you are familiar with Node. Download Node.js here then install it, and install node-static with one command line

$ npm install -g node-static

The usage of node-static is simple. Just enter the "dist" directory with command line, and type

static

It'll return something like this:

serving "." at http://127.0.0.1:8080

Then you can visit the template on http://127.0.0.1:8080 with your browser

Ready to Go...

Please, follow these steps to start admin template development

  • Install Node.js (NPM comes along with it)
  • Install Yeoman (if you are using npm 1.2.10 or above, this will also automatically install Bower, Grunt) for modern workflow.
  • Install Ruby, Sass , Compass for editing Sass CSS.

And you'll need to know how to use them, click the links to learn more if you not familiar with these yet. If it's your first time using these tools, it's probably too complex or even overwhelming to you. You'd better find someone to help you.

If you already know how to use them, then just do a few things to start developing Admin Template.

  1. Enter "Adminer" folder with your "Command Prompt" command line tool.
  2. npm install :- This command installs a Grunt plugins, and any packages that it depends on in the local node_modules folder.
  3. bower install :- This command installs a bower packages in the local bower_components/ folder.
  4. Then type grunt server and you are ready to go.

All the files for development are under ./Adminer/client/ folder.

Basic Template Structure

The basic structure of template (index.html) is quiet simple. It contains 3 parts: The header, which load all the stylesheets. The body, which contains the content of App. The footer, which load all the scripts.

The Header
